Review ALL code in this project (not just recently written code) and apply the improvements described below by editing the source files directly. Work incrementally — read a few related files, make your changes, then move on to the next area. Do not try to read the entire codebase before making any changes. If the code is already in good shape for a given check, it is fine to make no changes — but when you do find something worth improving, implement the fix in the code rather than writing a report or recommendation. Any guidance in a generic check below is subordinate to those project rules. Do NOT make changes that create large diffs for marginal improvement. Do NOT add docstrings, comments, or type annotations to code you didn't otherwise change — only document code you are actively modifying, and only when the change makes the code harder to understand without explanation. Do NOT change data schemas, database/search-engine mappings, document field types, or serialization formats — changing a field from str to int or adding/removing fields breaks existing indexed data and causes bulk index errors in tests. Every change should be clearly justified — if in doubt, leave the existing code alone. IMPORTANT: Do NOT run 'git push' or push commits to any remote.
